切换镜像源 Registry
为了加快包的下载和使用，改善使用体验，需要切换镜像源
更多信息，请访问 npmmirror
1. 查看 get - 安装镜像源，默认是：
> npm config get registry
https://registry.npmjs.org
2. 设置 set - 为淘宝最新镜像源 - 感谢22软工刘钰钿同学及时反馈
> npm config set registry https://registry.npmmirror.com
3. 再次查看，确认镜像源已更改
> npm config get registry
https://registry.npmmirror.com
开放脚本执行权限
以管理员方式打开PowerShell
查看权限 - 默认是禁止：Restricted
> get-executionpolicy
Restricted
修改权限为远程 remotesigned，按Y确认
> set-executionpolicy remotesigned
再次查看权限，确认修改成功
> get-executionpolicy
RemoteSigned
依赖 Dependencies
开发过程中，除了自己封装模块外，更多的是使用第三方|依赖
Node自带的NPM就极大的方便了开发人员的使用 - 需要哪个就安装哪个；不需要了就卸载掉
安装依赖 Install
install 可以简写为 i
默认是生产环境
1. 生产环境依赖 dependencies：项目上线后，所必须的依赖，如网络请求 axios；使用 --S / --save 命令行参数
> npm install package_name --save
> npm install package_name --S
2. 开发环境依赖 devDependencies：开发过程中需要的依赖，发布上线后，不再需要，如CSS预处理；使用 --D / --save-dev 命令行参数
> npm install package_name --save-dev
> npm install package_name --D
安装时，指定任何参数 --S / --save / --save-dev / D 等，会更新对应的 package.json 文件
如果不指定，则不更新；适合临时使用
npm 5+，--save 参数可以省略，默认在安装包时自动将依赖添加到 package.json 文件中的 dependencies 字段
安装时，使用命令参数 -g，可以全局安装依赖，供所有项目使用
移除依赖 uninstall
> npm uninstall package_name
