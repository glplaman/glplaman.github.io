猴子吃桃。猴子有若干个桃子。第一天吃了一半多一个；第二天吃了剩下的一半多一个；以后每天都吃剩下的一半多一个；第七天只吃了一个。问猴子共有多少桃子
从后往前推
第七天只有1个桃子，是第六天剩下的一半少一个。所以 +1再乘2就是第6天的桃子个数：(1 + 1)*2 = 4
第六天桃子个数的是第五天的一半少1。所以第五天的桃子个数：(4 + 1)*2 = 10
同理，第四天的桃子个数：(10 + 1)*2 = 22
同理，第三天的桃子个数：(22 + 1)*2 = 46
同理，第二天的桃子个数：(46 + 1)*2 = 94
同理，第一天的桃子个数：(94 + 1)*2 = 190

查找 search
顺序查找
从序列的一端开始，依次将记录的关键字和给定值进行比较：如果相等，查找成功；若查找整个表后，没有匹配的记录，查找失败
最少查找次数1；最多查找次数n；平均查找次数|比较长度 ASL = (n + 1)/2
适用于数据量小的情况；当序列特别多时，不适合顺序查找
时间复杂度 O(n)
更多内容，请访问 顺序查找
二分查找法
将序列看作一个区间，用low、high，mid = (low + high)/2 标识区间的下界、上界和中间位置；mid取整除结果
如果给定值小于中间记录值，则在前半段继续查找，high = mid - 1，low不变
如果给定值大于中间记录值，则在后半段继续查找，low = mid + 1，high不变
如果比较到最后，low > high ，查找结束
适合|必须有序表
查找范围每次都缩小一半，所以查找效率高
时间复杂度 O(lgn)
更多内容，请访问 二分查找法
