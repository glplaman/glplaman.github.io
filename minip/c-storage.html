Image

  • 按照一定规则命名，可以从0开始，如 0.png
  • 为了改善用户体验，应该准备不同尺寸的图；通常需要准备2种尺寸：缩略图和普通大图
  • 创建单独的文件夹保存图像文件，如 peike
  • 每个文件夹有自己的云路径
  • 数据库中图像文件对应的地址字段可以是部分路径，如 /peike/0.png，最后通过拼接文件夹的云路径，成为图像文件完整的云ID
  • 不建议直接拼接
    • <image src="cloud://cloud1-9gfr2z2oaabfeb0f.636c-cloud1-9gfr2z2oaabfeb0f-1259726622{{product.imageThumbnail}}" mode="aspectFill"/>
  • 拿到数据并处理，完成拼接
    • products.map(item => {
  item.image = 'cloud://cloud1-9gfr2z2oaabfeb0f.636c-cloud1-9gfr2z2oaabfeb0f-1259726622' + item.image
  item.imageThumbnail = 'cloud://cloud1-9gfr2z2oaabfeb0f.636c-cloud1-9gfr2z2oaabfeb0f-1259726622' + item.imageThumbnail
})
    <image src="{{product.imageThumbnail}}" mode="aspectFill"/>
